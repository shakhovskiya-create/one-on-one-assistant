# AI_SAFE_CHECKLIST.md
# EKF Hub — Обязательный чек-лист перед любым изменением

## 1) Что обязательно прочитать

- README.md
- CONTEXT.md
- ARCHITECTURE.md
- IMPROVEMENTS.md (если есть)
- DOMAIN_GLOSSARY.md (если есть)
- STATE_MACHINES.md (если есть)
- DECISIONS.md (если есть)
- TESTING.md (если есть)

## 2) Архитектурные проверки

- Бизнес-логика не попала в handlers/ws
- Нет implicit state transitions
- Нет дублирования правил между слоями
- Connector не владеет доменной логикой/состоянием

## 3) Security проверки

- Нет секретов/паролей в коде/compose
- Не открыты наружу DB/backend/frontend порты
- TLS verify не отключён в production
- Нет логирования токенов/паролей/секретов
- Есть валидация входных данных

## 4) QA/Testing проверки

- Есть Test Plan по изменению (scope + риски + проверки)
- Прогнаны статические проверки/тесты/смоук (или честно указано почему нельзя)
- Нет заявления "готово", если проверки не описаны или не выполнены
