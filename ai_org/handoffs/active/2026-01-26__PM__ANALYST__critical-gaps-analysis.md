## Зафиксированные замечания (12 пунктов)

### 1. Dashboard — неправильный sidebar
**Факт:** В левом sidebar размещены: сообщения, почта, Confluence, GitHub, настройки
**Требование:** По утверждённому макету эти элементы должны быть СВЕРХУ
**Расхождение:** Прямое нарушение согласованного дизайна
**Вопрос:** Кто принял решение изменить расположение? На каком этапе?

### 2. Tasks — отсутствует контекстный GitHub
**Факт:** GitHub отсутствует в sidebar раздела Задачи
**Факт:** GitHub почему-то есть в Dashboard, где он НЕ нужен
**Требование:** GitHub должен быть в контексте задач (связанные коммиты, PR)

### 3. Tasks — нет полной связи с проектами
**Факт:** Задачи не имеют обязательного контекста проекта
**Требование:** Проект — ОБЯЗАТЕЛЬНЫЙ атрибут задачи

### 4. Проекты — отсутствует вложение документов
**Факт:** В проектах нельзя прикрепить документы
**Требование:** Документы должны быть частью проекта

### 5. КРИТИЧНО: Заявка на улучшение — НЕ РЕАЛИЗОВАНА
**Факт:** Сущность полностью отсутствует в системе
**Требование:** Полный workflow:
1. Создание заявки
2. Процесс согласования
3. Формирование обоснования
4. Вынесение на комитет
5. Получение бюджета
6. Создание ПРОЕКТА

**Проект должен содержать:** бюджет, сроки, ответственный, статус, этапы

### 6. КРИТИЧНО: Планирование ресурсов — НЕ РЕАЛИЗОВАНО
**Факт:** Отсутствует полностью
**Требование:** В задаче должно быть видно:
- Кто исполнитель
- Сколько времени уже занято
- Сколько свободных часов в месяц
- Общая загрузка ресурса

### 7. БАГ: Зависимости задач — зависание UI
**Факт:** При открытии задачи интерфейс зависает на зависимостях
**Симптомы:**
- Блокировка задачи
- Постоянная загрузка
- Карточка не открывается
**Приоритет:** CRITICAL defect, блокирует работу

### 8. Meetings — НЕ соответствует макету
**Факт:** Текущая реализация принципиально отличается от утверждённого макета календаря
**Вопрос:** Почему согласовывается один макет, а реализуется другой?

### 9. СИСТЕМНАЯ ПРОБЛЕМА: Несоответствие макетов и реализации
**Факт:** Повторяется во ВСЕХ разделах: Tasks, Meetings, Service Desk, навигация
**Пример из макета Tasks (которого нет в реализации):**
- Тестирование
- Документация
- Требования
- Спринты
- Roadmap

### 10. КРИТИЧНО: Service Desk — НЕ РЕАЛИЗОВАН
**Факт:** Раздел полностью отсутствует
**Факт:** Был согласован, был в макетах
**Требование:** Критичен для портала

### 11. Почта — нет SSO
**Факт:** Требуется повторный вход через учётные данные
**Требование:** Сквозная аутентификация (SSO)
**Тип:** Архитектурная ошибка

### 12. Профиль — дублирование и неполное ФИО
**Факт:** Отображается только фамилия (было ФИО)
**Факт:** Профиль дублируется в топ-баре и sidebar
**Требование:** Полное ФИО, без дублирования

## Задание для Analyst

### Обязательные deliverables:

1. **`__questions.md`** — вопросы к требованиям и процессу
2. **`__fm.md`** — функциональная модель (как ДОЛЖНО быть)
3. **`__spec.md`** — спецификация расхождений (факт vs требование)

### Что зафиксировать:

1. Полный список расхождений между макетами и реализацией
2. Недостающие сущности (Заявка на улучшение, Service Desk)
3. Недостающий функционал (планирование ресурсов)
4. Критические дефекты (зависимости, зависание)
5. Архитектурные ошибки (SSO)
6. UX/UI расхождения

### Запрещено:

- Оправдывать текущую реализацию
- Предлагать "компромиссы"
- Игнорировать замечания
