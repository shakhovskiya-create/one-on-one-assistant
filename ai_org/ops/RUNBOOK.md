# Runbook: как ты работаешь с AI-командой (одна точка входа)

Ты общаешься только с PM (Product Manager). Остальные роли вызываются PM-ом.

## Цикл 1 задачи (идеальный)
1) Ты пишешь запрос PM (в чате/в handoff) — цель, контекст, ограничения.
2) PM обновляет: /WORKPLAN.md (или создает запись) и ai_org/state/STATUS.md.
3) PM создает handoff аналитику → аналитик пишет/обновляет требования (requirements / уточнения в md).
4) PM создает handoff дизайнеру (если нужно) → дизайнер фиксирует UX/UI решения.
5) PM создает handoff разработчику → разработчик делает код + обновляет архитектурные заметки при необходимости.
6) PM создает handoff QA → QA прогоняет тесты (и UX/UI чек) и пишет замечания.
7) PM принимает результат:
   - либо отправляет на доработку (новый handoff dev),
   - либо готовит релиз (handoff release manager / devops).
8) Release manager фиксирует релиз-заметки, DevOps делает деплой (по правилам безопасности).

## Главное правило контекста
- Любые решения: в /DECISIONS.md
- Любые изменения плана: в /WORKPLAN.md
- Любые риски/вопросы: в соответствующих местах (или отдельный файл, если нужно)
- Любая работа оставляет след в ai_org/logs/AGENT_LOG.md
